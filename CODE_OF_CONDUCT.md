# Code of Conduct

## Our Standard

Participation in this project should be respectful, direct, and constructive. The goal is to improve the software, not win arguments.

Examples of acceptable behavior:

- Giving actionable technical feedback
- Respecting different experience levels
- Assuming good intent while still being precise about problems
- Keeping discussions focused on the code, docs, and user impact

Examples of unacceptable behavior:

- Harassment, personal attacks, or intimidation
- Publishing someone else's private information
- Discriminatory or abusive language
- Deliberately disruptive behavior in issues, pull requests, or discussions

## Enforcement

Project maintainers may edit, hide, lock, or remove comments and contributions that violate this code of conduct.

For sensitive conduct issues, avoid public escalation when possible. Open a private channel with the maintainers once the repository is published, or use the security reporting path for issues involving private data exposure.
